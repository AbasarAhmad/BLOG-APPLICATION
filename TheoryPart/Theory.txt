1. Project Overview
	This project is a RESTful Blog Application backend developed using Spring Boot.
	It provides secure APIs for managing users, posts, categories, comments, and images.

  Key Features:
	User registration and login using JWT authentication
	Role-based authorization (ADMIN / NORMAL user)
	CRUD operations for Users, Posts, Categories, and Comments
	Pagination, sorting, and search functionality
	Image upload and download support
	Centralized exception handling
	Stateless and secure architecture

4. Entity and Database Design
	Core Entities:
 User
	Fields: id, name, email, password, about
	Implements UserDetails for Spring Security
	Relationships:
	One User ‚Üí Many Posts
	Many Users ‚Üî Many Roles

 Role
	Fields: id, name
	Used for authorization (ADMIN, NORMAL)

 Category
	Fields: title, description
	One Category ‚Üí Many Posts

 Post
	Fields: title, content, imageName, addedDate
	Relationships:
	Many Posts ‚Üí One User
	Many Posts ‚Üí One Category
	One Post ‚Üí Many Comments

 Comment
	Fields: content
	Many Comments ‚Üí One Post
	All relationships are mapped using JPA annotations.

5. DTO (Data Transfer Object) Layer
 Why DTOs?
	Prevent direct exposure of entities
	Avoid security issues (e.g., password leakage)
	Prevent circular reference problems
	Keep API responses clean and controlled

Examples:   UserDto,PostDto,CategoryDto,CommentDto, PostResponse (for pagination)

6. Authentication Flow (JWT Based)
	Login API
	POST /api/v1/auth/login
  Flow:
	User sends email and password
	AuthenticationManager authenticates the user
	CustomUserDetailService loads user from the database
	Password is verified using BCrypt
	JWT token is generated
	Token is returned to the client

7. JWT Token Working
    Token Contains:
	Subject (user email)
	Issue time
	Expiration time
	Signed using HS512 algorithm

    Why JWT?
	Stateless authentication : Stateless authentication means the server doesn‚Äôt store session info; the token itself carries everything 						needed for authentication.
	No server-side session storage
	Scalable and secure

8. Request Authorization Flow
   For protected APIs:
	Client sends request with header:
	Authorization: Bearer <JWT_TOKEN>
	JwtAuthenticationFilter:
	Extracts token
	Retrieves username
	Loads user details
	Validates token
	Authentication is set in SecurityContextHolder
	Request is allowed to access controller
	‚ùå Invalid or missing token ‚Üí 401 Unauthorized

9. Role-Based Authorization
	Roles are stored in the database
	User roles are converted to authorities
	Method-level security using annotations

Example:  @PreAuthorize("hasAuthority('ADMIN')")
	Only ADMIN users can delete users.

10. Security Configuration Summary
	CSRF disabled (JWT-based REST API)
	Stateless session management
  Public APIs:
	/api/v1/auth/**
  All GET requests
	Secured APIs:
	POST / PUT / DELETE requests
	JWT filter added before Spring Security authentication filter
